# Aircraft Optimizer Platform

This folder is the master folder for the aircraft optimizer platform for small unmanned aircraft.

The platform is intended to become a standalone, traceable, extensible aircraft design and optimization ecosystem. It should eventually contain or vendor the SDF generation software, manual and automatic aircraft definition paths, STL exporter adapters, optimizer core, analysis modules, dashboard, schemas, run records, and artifacts.

## Scope

Initial scope:

- Small UAVs.
- Fixed-wing aircraft.
- Loiter/endurance optimization.
- Continuous design variables only.
- Geometry generated by the existing implicit CAD/SDF aircraft modeling system.

Future scope:

- VTOL and tailsitter families.
- Multi-mission optimization.
- Structural optimization.
- Mission simulation.
- Surrogate-model-assisted optimization.
- MDAO workflows.
- Selective high-fidelity CFD.

Explicit early exclusions:

- No production optimizer code yet unless a later task starts implementation.
- No redesign of the CAD/SDF geometry system.
- No initial topology changes.
- No initial aircraft-family changes.
- No default full-CFD campaign execution.

## Platform Principles

- Treat traceability as a core feature, not an afterthought.
- Keep geometry, analysis, scoring, optimization, logging, artifacts, and dashboard concerns decoupled.
- Preserve failed candidates.
- Make reruns and rescoring append-only.
- Use escalating fidelity and avoid expensive CFD when cheap gates are sufficient.
- Treat batch sweeps as secondary diagnostic/campaign tools; the primary autonomous loop is sequential and gated.
- Keep every major subsystem replaceable.
- Store enough metadata for future reproducibility, ML datasets, and MDAO workflows.

To import the current five-variant snappy v0.4 evidence package:


This imports existing mesh/solver artifacts only. It does not rerun meshing or
launch OpenFOAM. The legacy `scoring_allowed=false` field means final scoring is
blocked; rough scoring can still be enabled by the named steady policy. The
import also persists surface-fidelity reports and, when present, bounded no-slip
kOmegaSST force coefficients plus `yPlus` fields as qualified scoring evidence.
